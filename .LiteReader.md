# Kotlin Android EPUB Reader 引入 Rust 核心引擎改造方案

## 目标

当前 Android EPUB 阅读器采用 Kotlin EPUB 解析方案。

存在问题：

- 大型 EPUB（50MB~500MB）打开慢
- 解析整本书导致启动阻塞
- HTML/XHTML解析耗时
- 图片资源加载慢
- 内存占用高
- 章节数量多时初始化时间过长

改造目标：

引入 Rust 作为 EPUB 核心解析引擎。

最终架构：

Android Kotlin:
- UI
- Compose
- 用户交互
- 生命周期
- LIBGEN搜书（源码保持不变）
- 下载管理
- 阅读记录

Rust:
- EPUB解析
- ZIP读取
- XML解析
- HTML解析
- CSS解析
- 图片资源管理
- 搜索索引
- 缓存


目标性能：

|项目|目标|
|-|-|
|打开100MB EPUB|<1秒显示第一页|
|章节切换|<100ms|
|图片加载|异步|
|全文搜索|毫秒级|
|内存占用|降低50%以上|

```

↓

↓

```

```

↓

↓

↓

↓

```

导致：

```

Compose UI

↓

ViewModel

↓

Repository

↓

↓

↓

Compose Render


```

```

↓

Rust读取container.xml

↓

↓

↓

↓

↓

```

# 四、EPUB解析流程


Rust负责：


## 1. 打开文件


输入：

```
book.epub

```


不解压。


使用：

```
ZIP random access

```


读取：

```
META-INF/container.xml

```


获得：

```
content.opf

```



---

## 2. OPF解析


解析：

```
metadata

manifest

spine


```


生成：

```rust

struct Book {

title:String,

author:String,

cover:String,

chapters:Vec<ChapterIndex>

}


```


---

## 3. 建立章节索引


不要解析章节内容。


只保存：


```rust

struct ChapterIndex {

id:String,

title:String,

href:String,

order:i32

}


```


返回 Kotlin。

# 开发原则


1. EPUB永远不全部解析

2. 所有资源Lazy Load

3. Rust负责计算

4. Kotlin负责显示

5. 页面缓存优先

6. 图片按需读取

7. 不使用WebView作为阅读核心

```
